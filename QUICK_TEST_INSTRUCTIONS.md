# Быстрый тест одной модели

Для быстрой проверки работы стратегии после изменений:

## Вариант 1: Использовать упрощенный скрипт

```bash
python test_single_model.py
```

Этот скрипт:
- Автоматически найдет первую доступную модель для BTCUSDT
- Запустит бэктест на 30 днях
- Выведет подробные результаты

## Вариант 2: Использовать compare_ml_models.py для одной пары

```bash
python compare_ml_models.py --days 30 --symbols BTCUSDT
```

## Что проверить в результатах:

1. **Количество сделок:** должно быть 30-100+ за 30 дней (вместо 1-2)
2. **Win Rate:** должен быть > 0% (хотя бы некоторые сделки прибыльные)
3. **Long/Short баланс:** должны быть и LONG, и SHORT сигналы
4. **Avg Trade Duration:** должно быть < 168 часов (сделки должны закрываться по TP/SL, а не по таймауту)
5. **TP/SL в отладочном выводе:** проверьте, что TP и SL правильно рассчитываются

## Если проблемы остаются:

1. **Мало сигналов:** еще больше снизить пороги (threshold_mult до 0.10-0.15)
2. **Все SHORT:** проверить данные и логику предсказаний
3. **Все по таймауту:** проверить расчет TP/SL (должны быть в пределах 0.5-3% от цены)
